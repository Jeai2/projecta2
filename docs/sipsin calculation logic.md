# 십신 계산법 명세서 (Sipsin Calculation Specifications)

> **목적 (Purpose):**
> AI API 및 개발자가 명리학(십신) 데이터를 처리할 때 참조할 수 있도록 계산 로직, 전제 조건, 제약 사항을 명확히 정의합니다.

---

## 1. 전제 조건 (Prerequisites)
<Prerequisites>

### 1.1. 기초 요소 분류 (Base Element Classification)
- **길신 그룹 (Gilsin Group):** 비견, 식신, 정재, 편재, 정관, 정인
- **흉신 그룹 (Hyungsin Group):** 겁재(양인), 상관, 편관(칠살), 편인(효신)
  *※ 별칭: 겁재는 양인(羊刃), 편관은 칠살(七殺), 편인은 효신(梟神)으로 혼용되어 표기될 수 있음.*

### 1.2. 그룹 분류에 대한 해석적 제약 (Interpretation Constraints on Grouping)
- **Constraint (제약):** 길신/흉신의 레이블링은 정통 명리학의 구조적 분류 체계일 뿐, 실제 사주 작용에서의 '좋고 나쁨(Positive/Negative)'을 절대적으로 의미하지 않는다.
- **Rule (규칙):** AI는 해석 로직을 수행할 때 `길신 = 무조건 긍정적 결과`, `흉신 = 무조건 부정적 결과` 형태의 이분법적 수치 연산이나 감성 분석(Sentiment Analysis)을 절대 금지한다.
- **Reasoning (사유):** 흉신 모델이더라도 적절히 제화(制化)되거나 환경에 맞으면 강한 돌파구와 큰 성취(몸에 좋은 쓴 약)를 만들어내며, 길신이더라도 과다하거나 쓰임이 엇갈리면 흉하게 작용할 수 있기 때문.

### 1.3. 십신 1:1 기초 생극 관계 (Basic 1:1 Saeng/Geuk Relationships)
- **설명:** 십신 간의 1:1 생(生)과 극(克) 작용 시 발생하는 가장 기초적인 길흉(Good/Bad) 판별 기준.
- **예외 처리:** 앞선 `1.2 제약 사항`에 따라, 이 테이블의 Good/Bad는 최종 결과가 아닌 '해당 작용 자체의 성격'을 나타내는 기초 데이터로 사용된다.

| Source (주체) | Action (작용) | Target (객체) | Condition (조건) | Result | 논리적 사유 (추론용) |
|---------------|-------------|-------------|----------------|--------|--------------------|
| 모든 십신     | 生 (생)      | 상생되는 십신 | -              | Good   | 생해주는 작용은 기본적으로 순리이므로 긍정적 |
| 비견/겁재     | 克 (극)      | 정재/편재     | 일간 신강(Strong)| Good   | 내 힘이 강할 때 재물을 장악(통제)할 수 있음 |
| 비견/겁재     | 克 (극)      | 정재/편재     | 일간 신약(Weak)  | Bad    | 내 힘이 약한데 재물을 취하려다 화를 입음 (군비쟁재 등) |
| 식신/상관     | 克 (극)      | 정관          | -              | Bad    | 올바른 관(법/질서)을 상하게 함 (상관견관 등) |
| 식신/상관     | 克 (극)      | 편관(칠살)    | -              | Good   | 어려움/살(殺)을 능동적으로 해결함 (식신제살 등) |
| 정재/편재     | 克 (극)      | 정인          | -              | Bad    | 바른 문서/도덕성을 재물욕이 훼손함 (탐재괴인) |
| 정재          | 克 (극)      | 편인          | -              | Bad    | 정재가 편인을 극하면 부정적 고뇌 유발 |
| 편재          | 克 (극)      | 편인          | -              | Good   | 치우친 생각(편인)을 현실감각(편재)으로 깨우침 |
| 정관          | 克 (극)      | 비견          | -              | Bad    | 나의 형제/동료를 지나치게 억압함 |
| 정관          | 克 (극)      | 겁재          | -              | Good   | 나의 재물을 탐하는 도둑(겁재)을 합리적 규칙(정관)으로 통제함 |
| 편관          | 克 (극)      | 비견/겁재     | -              | Bad    | 살(殺)이 나와 내 동료의 힘을 무자비하게 억압함 |
| 정인          | 克 (극)      | 식신          | -              | Bad    | 생각(정인)이 지나쳐 행동/밥그릇(식신)을 엎음 |
| 정인          | 克 (극)      | 상관          | -              | Good   | 제어 안되는 반항심(상관)을 바른 생각(정인)으로 교화함 (상관패인) |
| 편인          | 克 (극)      | 식신/상관     | -              | Bad    | 부정적 사고(편인)가 나의 행동력/밥그릇을 훼손함 (편인도식) |

### 1.4. 계산 작용 범위 및 상호작용 규칙 (Interaction Scope & Mechanics)
<Rules>
- **Rule 1. 간지(干支) 분리 연산 원칙:** 
  - 천간은 천간끼리만 연산하며, 지지는 지지끼리만 연산한다. (동일 레이어 병렬 연산)
  - 천간(순수한 기운)과 지지(지장간이 포함된 혼잡 기운)는 서로 직접적인 연산(결합/상극) 영향을 주지 않는다.

- **Rule 2. 원국 천간의 광역 작용 (Non-local Interaction):**
  - 사주 원국 내의 천간은 기둥(년, 월, 시간) 상의 거리 제약 없이 상호작용이 가능하다. 
  - *EX)* 년간이 시간에, 혹은 시간이 년간에 직접적인 영향을 미칠 수 있다.

- **Rule 3. 원국 지지의 인접 작용 (Local Interaction):**
  - 사주 원국 내의 지지는 위치가 고정되어 있으므로, 오직 **맞닿은 양옆 기둥의 지지**와만 상호작용한다.
  - `년지 ↔ 월지 ↔ 일지 ↔ 시지` 구조를 띠며 기둥을 건너뛰어 작용하지 못한다. (예: 년지는 일지와 직접 영향 불가)

- **Rule 4. 운(運)의 연산 개입 (Daewoon & Seowoon Scope):**
  - 대운, 세운(년운), 절운(월운), 일진(일운) 역시 간지 분리 원칙(천간은 천간과, 지지는 지지와 연산)을 동일하게 따른다.
  - 단, 외부에서 들어오는 **운(運)의 지지는 원국 지지와 달리 거리 제약을 받지 않으며**, 사주 원국의 4개 지지(년지, 월지, 일지, 시지) 모두에게 동시다발적으로 영향을 행사한다.

- **Rule 5. 일간(日干) 지연 평가 (Lazy Evaluation of Day Master):**
  - 일간(日干)은 사주 전체의 기준점이다. 따라서 나머지 7개 글자 및 운의 간지에 대한 모든 생·극·합·충 연산이 **완전히 종료된 후, 가장 마지막에 대입**하여 최종 상태(신강/신약, 길흉 판단 등)를 확정한다.
</Rules>

### 1.5. 수호 대상(Core Sipsin/용신) 설정 및 훼손 검증
<Rules>
- **Rule 1. 코어 십신 찾기:** 천간과 지지에서 각각 일간(나)에게 가장 핵심적으로 도움이 되는 십신(용신/상신)을 설정한다. 
- **Rule 2. 방어/보호 우선 원칙:** 설정된 코어 십신은 철저히 보호되어야 한다. "보호한다"의 의미는 해당 십신을 극(克)하려는 적대적 글자의 접근을 타 요소(상신, 통관 등)를 이용해 막아내는 것을 뜻한다. (예: 정재가 코어일 때, 재물을 노리는 겁재의 접근을 차단해야 함)
- **Rule 3. 훼손 여부 판별 로직:** 
  - **천간:** 코어 십신이 합반(기능 정지), 합거(사라짐), 혹은 극(克)을 방어해내지 못하면 훼손된 것으로 본다.
  - **지지:** 코어 십신이 형·충·파·해의 물리적 충돌에 노출되어 깨지면 훼손된 것으로 본다.
- **Rule 4. 훼손 시 해석:** 코어 십신이 훼손된 것으로 판독되면, 해당 코어 십신이 매핑되는 '육친(가족, 대인관계, 사회적 포지션, 재산 등)' 영역에 심각한 문제가 발생하는 것으로 최종 해석 스크립트를 반환한다.
</Rules>

### 1.6. 생극의 흐름(Flow Trace) 및 The Third Wheel 알고리즘
<Rules>
- **Rule 1. 기운의 연쇄 사슬 추적:** 천간은 천간끼리, 지지는 지지끼리(특히 인접성을 기반으로) 서로 생(生)해주는 흐름의 파이프라인(`A 生 B 生 C`)을 구성한다.
- **Rule 2. 종착지 영향력 판별 (Impact Analysis):** 사슬이 끝나는 최종 종착점(Terminal Node)에 있는 십신이 최종적으로 일간을 향해 어떤 작용(생하여 돕는지, 극하여 괴롭히는지)을 하는지가 사주 전반의 길흉 체감도를 결정한다.
- **Rule 3. "The third wheel" (고립된 구원자) 특수 판별 로직:**
  - **Capability (해결 수단 보유 판독):** 사슬의 종착지가 일간을 괴롭히는 흉신(예: 편관)일 때, 사주 내 어딘가에 이를 완벽히 타격하여 제압할 수 있는 무기(예: 식신제살이 가능한 식신)가 존재하는지 스캔한다.
  - **Reachability (물리적 도달 불가 판독):** 무기(식신)가 시지(時支) 등에 박혀 있어, 지지의 인접성 원리(국소성 제한)에 의해 타격 대상(예: 월지의 편관)까지 도달할 수 없는 상황임을 확인한다.
  - **Result (최종 판단 및 해석):** 위 조건을 모두 만족하는 해당 무기(식신)를 가리켜 **"The third wheel"** 상태로 규정한다. 이는 '무기가 없어 당하는 것'이 아니라 **'최고의 무기가 있으나 물리적으로 단절되어 작동하지 못하는 멍청이가 된 로봇 상태'**를 뜻하며, 불필요한 혼잡과 심리적/현실적 좌절감이 극대화된다는 특수 통변 텍스트를 출력하도록 한다.
</Rules>

### 1.7. 병렬 연산 및 하드 카운터 방어(Shield) 특수 알고리즘
> _**[System Prompt for LLM Parsers (Claude, Gemini, etc.)]**_
> 다음 규칙을 활용하여 단순 자연어적 수치 연산 오류(HP 덧셈/뺄셈 방식)를 원천 차단하고 구조화된 로직(Pseudo-code)으로 십신 간 상호작용을 계산할 것.

<Rules>
- **Rule 1. 천간의 병렬 연산 (Simultaneous Conflicts):** 
  - 천간은 거리 제한이 없으므로 모든 생·극 연산이 **동시다발적**으로 처리된다.
  - `A 生 B 生 C` 의 매개 파이프라인이 존재하더라도, 천간의 비국소성 원리에 의해 `A`가 `C`를 타격하는 `A 克 C` (상관견관 등) 현상은 **무효화되지 않으며 동시에 발생**한다.
- **Rule 2. 운 지지의 광역 쉴드 (Global Shield of Daewoon/Seowoon):**
  - 명조 원국의 지지 간 작용은 '인접 규칙(Local)'에 묶여 있으나, **대운/세운 등 외부 운(運)에서 유입된 지지 글자는 위치적 한계를 벗어나 사주 전체(Global)에 동시다발적으로 개입(방어 및 파괴)**할 수 있다.
- **Rule 3. 방어의 절대 조건 (Zero-Defense of 'Saeng'):**
  - 특정 십신(T)이 외부 세력(A)에게 공격(克)당할 때, T를 생(生)해주는 우호 세력은 방어력(Shield)에 기여하지 못한다 (결론: 생은 방어력이 0이다). 
  - 특정 십신(T)을 방어(보호)하기 위한 유효한 로직은, 오직 **적(A)을 직접 극(克)하여 통제하는 수호자(D)의 존재 여부**뿐이다. (`Defender 克 Attacker`)
- **Rule 4. 하드 카운터 예외 로직 (Hard-Counter Logic):**
  - `IF Action == "克" AND Source == "상관" AND Target == "정관":`
    - `WARNING:` Target에 대한 무의미한 생(生) 버프(단순 덧셈 연산)를 완전히 무시할 것.
    - `CONDITION:` Source(상관)을 제압(克)하는 Defender(정인)가 존재해야만 Target(정관)이 안전하다 판정함. (*상관패인 로직*)
    - `FAIL:` Defender가 없을 경우 정관은 완전히 파괴된(파격) 것으로 치명상 판정 수행.
  - 특정 십신 간에는 고유 명칭(예: 상관=관을 상하게 함)에서 알 수 있듯 **태생적 카운터 관계**가 존재하므로, 이를 단순 에너지 상쇄 계산으로 퉁치려는 LLM의 Hallucination을 절대 금지함.
</Rules>
