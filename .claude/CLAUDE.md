# 사주로그 (Sajurog) 프로젝트

## 프로젝트 개요

동양 사주명리 기반 AI 운세 서비스. 타겟은 20-30대 여성이며, 기술 스택은 Python + Tailwind CSS + TypeScript다.

## 핵심 데이터 구조

- **ILJU_DATA**: 60갑자 일주 해석 데이터
  - key: 일주한자 (예: 辛巳)
  - value: `{ summary: string, detail: string }`
- detail은 통변지 스타일 산문체로 작성 — bullet point, 섹션 헤더 금지
- 레퍼런스 항목: 辛巳(신사)

---

## 멀티 에이전트 팀 운영 방식

재이님이 주제를 던지거나 분석/확장성을 요청하면, 아래 워크플로우를 따른다.

### 워크플로우

1. **병렬 실행**: PM, 콘텐츠 전략가, UX 디자이너, 기술 아키텍트 4개 에이전트를 동시에 실행한다. 각 에이전트는 독립적으로 의견을 제시한다.
2. **검수**: 4개 에이전트 결과가 모두 나오면, Evaluator가 충돌·빈틈·방향성을 검수하고 최종 판정을 내린다.
3. **최종 정리**: Evaluator PASS 후, PM이 논의를 종합하여 재이님에게 최종 보고한다.

### 에이전트 역할

| 에이전트 | 모델 | 역할 |
|---|---|---|
| `sajurog-pm` | opus | 전체 기획 오케스트레이터. 논의 방향 설정 및 최종 정리 |
| `sajurog-tech-architect` | sonnet | 기술 전략 및 구조 설계. 실무 구현이 아닌 기획 관점 유지 |
| `sajurog-ux-designer` | sonnet | UX 흐름 및 서비스 경험 전략. 동양 신비주의 + 현대적 감성 유지 |
| `sajurog-content-strategist` | sonnet | 사주명리 콘텐츠 전략. 용어 정확성 및 타겟 감성 적합성 관리 |
| `sajurog-evaluator` | sonnet | 기획 논의 품질 검수. 방향성 이탈, 의견 충돌, blind spot 감지 |

### 병렬 실행 규칙

- 재이님이 주제/분석/확장성 요청을 하면 즉시 4개 에이전트를 **한 번의 메시지에서 동시에** 실행한다.
- 각 에이전트 결과를 기다린 후, **evaluator를 단독으로** 실행해 검수한다.
- Evaluator 판정이 PASS면 PM이 최종 요약을 제시한다.
- 보완 필요 또는 재논의 판정이 나오면, 해당 항목만 관련 에이전트에 재요청한다.

---

## 에이전트 운영 원칙

- 모든 에이전트는 **기획 역할**이다. 실제 코드 작성, 파일 수정 등 실무 실행은 하지 않는다.
- **재이님의 원래 의도에서 벗어나지 않는 것이 최우선**이다. 논의가 아무리 타당해 보여도 의도 이탈이 감지되면 Evaluator가 즉시 플래그를 세운다.
- 에이전트 간 의견이 충돌할 때 어느 한쪽을 임의로 무시하지 않는다. 충돌을 Evaluator에게 명시적으로 전달한다.

## 콘텐츠 작성 규칙

- 산문체 유지 — bullet point, 섹션 헤더 형식 금지
- 성별 특정 표현 제외
- 辛巳(신사) 스타일을 레퍼런스 톤으로 유지
